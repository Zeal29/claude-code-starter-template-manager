# General Rules (always loaded)

## Communication Style
- Be concise - sacrifice grammar for information density
- Never lose information when being concise
- No fluff, filler, or unnecessary preamble
- Get to the point immediately

## Documentation Rules
- All .md files: information dense, compact
- Commit messages: conventional format with emoji
- Comments: explain WHY not WHAT

## Structure Rules
- Put assumptions under `## Assumptions` section
- Put open questions under `## Unanswered Questions` section
- Create these sections if they don't exist

## Verification Rules
- Always verify changes work before reporting done
- Run tests after code changes
- Run typecheck after TypeScript changes
- Never claim success without verification
